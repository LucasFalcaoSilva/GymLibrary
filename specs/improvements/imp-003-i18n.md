## ErrorMapper Update

`ErrorMapper.kt` deve usar `Context` para retornar strings localizadas:

```kotlin
fun Throwable.toUserMessage(context: Context): String = when (this) {
    is UnknownHostException,
    is ConnectException -> context.getString(R.string.error_no_internet)
    is SocketTimeoutException -> context.getString(R.string.error_timeout)
    is HttpException -> when (code()) {
        401, 403 -> context.getString(R.string.error_auth)
        404      -> context.getString(R.string.error_not_found)
        in 500..599 -> context.getString(R.string.error_server)
        else     -> context.getString(R.string.error_client)
    }
    else -> context.getString(R.string.error_unknown)
}
```

## Business Rules

- BR-01: PT-BR é o idioma padrão — `res/values/strings.xml`
- BR-02: Nenhuma string hardcoded nas telas — todas via `stringResource()`
- BR-03: Nomes de exercícios, grupos musculares e instruções vêm da API — não traduzir
- BR-04: `ErrorMapper` deve receber `Context` para retornar mensagem localizada
- BR-05: Strings de erro de operação (paginação) também devem usar `stringResource()`
