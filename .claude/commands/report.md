# /report — Generate Implementation Report

Generate a markdown report for a completed spec implementation and save it to `reports/`.

## Usage

```
/report CORE-001
```

## Steps

1. Read the target spec file from `specs/`
2. Read `docs/coding-conventions.md`
3. List all files created or modified by this spec
4. Run `git log --oneline -1` to get the commit hash
5. Run `git show --stat HEAD` to get files changed in the commit
6. Generate the report following the format below
7. Save to `reports/SPEC-ID.md`
8. Confirm the file was created

## Report Format

```markdown
# Implementation Report — [SPEC-ID]

**Date:** YYYY-MM-DD  
**Spec:** specs/path/to/spec.md  
**Commit:** [hash] [message]  
**Build:** ✅ BUILD SUCCESSFUL / ❌ BUILD FAILED  

---

## Spec Summary

[One paragraph describing what this spec implements]

## Files Created

| File | Description |
|------|-------------|
| path/to/file.kt | what it does |

## Files Modified

| File | Change |
|------|--------|
| path/to/file.kt | what changed and why |

## Business Rules Compliance

| BR | Description | Status |
|----|-------------|--------|
| BR-01 | description | ✅ / ❌ |

## Deviations from Spec

| Deviation | Reason | Impact |
|-----------|--------|--------|
| description | why it was necessary | none / low / high |

If no deviations: "None — implementation matches spec exactly."

## Review Findings

| # | Severity | File | Description | Status |
|---|----------|------|-------------|--------|
| 1 | 🔴/🟡/⚪ | file:line | description | Fixed / Deferred to SPEC-ID |

If no findings: "No findings — all BRs passed on first review."

## Deferred to Next Spec

| Item | Target Spec |
|------|-------------|
| description | SPEC-ID |

If nothing deferred: "Nothing deferred."

---

*Generated by /report command*
```

## Rules

- Always include the commit hash — never leave it blank
- Deviations must explain WHY, not just WHAT changed
- Deferred items must reference the exact spec ID that will address them
- If build failed, mark ❌ and include the error summary
- Never fabricate BRs or findings — only report what actually happened
